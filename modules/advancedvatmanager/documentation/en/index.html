Advanced VAT Manager (EN)

Version 1.6.0

 

The Advanced VAT Manager module is a module specially designed to manage customers who have a VAT number valid for intra-community operations and to certain countries with import agreements (such as England and Norway) and eliminate the collection of taxes on their purchases. The validation of the European or UK VAT number is done through the official VIES or GOV.UK system.

Furthermore, it is a very complete module that includes many advanced configuration options. The most prominent are:

  • Eliminate tax charges for customers who have a valid VAT number set up through validation by format or via the VIES and GOV.UK API.
  • Updated with the Brexit regulations and the collection of taxes with the threshold of 135 pounds.
  • Updated with the VOEC regulations (Norway) and the collection of taxes to Norwegian consumers and tax exemption to companies.
  • Validation option for the Front-Office and Back-Office.
  • Choice of countries for VAT validation.
  • Choice of local country for tax collection.
  • VAT number formatting system (converts VAT numbers entered by customers to the correct format)
  • It shows the prices without taxes in the product sheets, product list, shopping cart and cart summary.
  • Automatic assignment of specific customer group for customers who register with a valid VAT number.
  • Automatic assignment of client group by country.
  • Tax exemption for customer groups.
  • Option to configure the VAT number field as optional or mandatory.
  • Validation of company name registered in the official system.
  • Customization of the label and description of the VAT nAutomatically insert the name of the company: If this option is activated, the system will extract the data of the company name that is registered in the official bodies VIES or GOV.UK, when the client inserts his VAT number and will save it in the field «company name». In the event that a client has entered a different company name than the one registered with official bodies, this will be replaced by the correct name. In this way, the customer’s address will always contain the name of the correct and officially registered company.umber field in the form.
  • Email alert system.
  • Address analysis system and verification of VAT numbers stored in the store’s database.

The Advanced VAT Manager module is the most complete module that has been developed to manage customer addresses for intra-community operations.

Minimum requirements

The requirements of the values in the parameters for this module to work correctly are the following:

Minimum requirements

PHP version 5.6.0
PHP extensión php_soap, php_curl

The module uses a SOAP-type webservice to connect to the VIES service in the VAT number validation process and uses the CURL extension to connect to the GOV.UK service in charge of validating UK VAT numbers. It is a minimum requirement to have the php_soap and php_curl extension installed on the server. Check with your hosting provider to install if necessary.

Recommended Requirements (For the use of the database address scanning system)

Memory limit (memory_limit) 512M
Maximum input time (max_input_time) 300 seconds or higher
Maximum execution time (max_execution_time) 300 seconds or higher

 

Installation

1. Upload the module

The first step in the module installation is to upload the module folder to the Prestashop platform. For this procedure, you have two options to follow:

From FTP

You have to unzip the module file and using FTP software such as FileZilla or similar, upload it to the Modules folder located in the root directory of the store.

From the Prestashop Back-office

Depending on the version of Prestashop installed (1.6 or 1.7) the procedure may vary a bit, however the principle of the process is the same.

You can find more information about this process at the following link:

https://addons.prestashop.com/en/content/13-installing-modules

Prestashop 1.6

You have to access the Modules and Services tab, click on Add a new module, then select the compressed file of the module that you have previously downloaded from the official page of Prestashop Addons, and click on Upload this module.

Once the upload process is finished, you will be able to view the module in the list of available modules and with the option to install.

Prestashop 1.7

In this case you have to access the Modules tab and click on Module Catalog. In the upper right corner you will find a button that says Install a module, click on it and you will be able to select or drag the compressed file of the module that you have downloaded from the Prestashop Addons page. You can install the module from the moment you have uploaded it or by accessing the list of modules in Module management within the same Modules section.

 

2. Installation

Once uploaded, you have to access the list of modules, search for the module and click on the INSTALL button

Once the installation process is finished, you can access the module configuration page by clicking on CONFIGURE

Configuration

 

Activation settings

  • Back-office validation: This option enables or disables the validation of VAT numbers in the Back-office address form and also from CRON tasks or driver requests executed from the admin part to import customers or orders.
  • Front-office validation: This option activates or deactivates the validation of VAT numbers in the Front-office address and checkout form.
    Countries for validation: Select the countries for which the system will perform the validation of VAT numbers. According to the legislative framework, they have to be countries that are within the European Union.
    This option comes predefined with all the countries of the European Union selected. Under normal conditions you should not modify this option.
  • Countries for validation: Select the countries for which the system will perform the validation of VAT numbers. According to the legislative framework, they have to be countries that are within the European Union and those that have special import agreements (Brexit, Norway…).
    This option is predefined. Under normal conditions you should not modify this option.
  • Disable validation for modules: Option to select third-party modules for synchronization/import of catalogs and orders from marketplaces, which you want to disable during the validation process of customer addresses and their VAT numbers. This option is very useful for those marketplace modules that import their own order amount data and manage the tax exemption from their platforms. In this way, the module does not interfere in the process of importing and synchronizing and validating the data. If the list is empty, it is because you do not have any marketplace module installed that needs to be disabled.

The United Kingdom country is also selected by default, since Northern Ireland belongs to the United Kingdom but is within the European Union, and United Kingdom must be selected in order to validate UK VAT numbers to comply with the option » Brexit» with the regulation of the collection of taxes at the threshold of 135 pounds.

General settings

  • Merchant VAT: Insert the merchant’s VAT number. This data is necessary for the activation of the 2-way validation type. For an invoice to be fully legal for tax exemption in intra-community transactions, both the merchant and the customer must have a valid VAT number registered. For this reason, 2-way validation is the safest and most correct way to validate intra-community transactions.
    In case of selecting the type of validation by 1 route, it would not be necessary to insert the VAT number in this option.

Advanced settings

  • Display warning messages in the address: Shows warning messages in the addresses section of the account when the client has created addresses that have not been validated yet or that have an invalid company or VAT number field. This serves to notify the client that they must modify these addresses. The warning messages depend on the options chosen within the module configuration page (VAT field, Company Validation…).


  • Do not allow checkout when the address is incomplete: Shows an error message during the purchase process so that the customer enters the address and updates the data when he has an incomplete address because he has not yet validated his VAT number, company or has a wrong value in these fields. The error messages depend on the options chosen within the module configuration page (VAT field, Company validation…).

    The customer will not be able to continue with the purchase process until the error in their addresses is resolved.
  • Custom Price Block: Option to display custom text on the product page [below the price] when a customer has a registered address with a valid VAT number and is exempt from paying taxes.

Validation settings

  • Validation system: Select the validation system for VIES or GOV.UK. The 1-way option validates the customer’s VAT numbers without verifying that the merchant has a valid VAT number. The 2-way option verifies that the merchant has a valid VAT number before verifying the customer’s VAT number. In the event that both have valid VAT numbers, then the validation process is completed successfully.
    To make use of the 2-way validation system, a valid merchant VAT number must be inserted in the «General Settings» section
  • Validation mode: You can choose between strict or smart validation system.
    Strict validation system: The system verifies that the format of the VAT number is correct and does not contain invalid characters. Example of invalid format: FR-25.458.473, FR 25 458 473, FR25.458.473
    Valid format example: FR25458473
    Intelligent validation system: The system validates the format of the VAT number even if it contains the invalid characters of the previous example. These characters will be removed and the VAT number will be extracted with its correct format. Example: FR-25.458.473 will be converted to the correct FR25458473 format.
  • Validation type: You can choose between «Format Validation» and «Format + API»
    Format validation: The system only validates VAT numbers according to their format in the numbering with their ISO code. But this type of validation does not perform the official validation through the validation API system, so it is not guaranteed that the inserted VAT number is registered in the VIES or GOV.UK system for intra-community operations.
    Format + API: The system performs a complete validation procedure, starting with the number format validation and followed by the official VIES or GOV.UK validation (if you have the Brexit option activated and it is a UK VAT number) . This option is the safest and most reliable to be able to validate the correct VAT numbers, ensuring that they are officially registered with VIES or GOV.UK.
  • Company Validation: If this option is enabled, the system will check the company name entered by the customer against the company name that is registered with the VIES or GOV.UK official bodies when the customer enters their VAT number . If they do not match, it will not allow you to continue with the address creation process.
  • Auto-insert company name: If this option is activated, the system will extract the data of the company name that is registered in the official bodies VIES or GOV.UK, when the client inserts his VAT number and will save it in the field «company name». In the event that a client has entered a different company name than the one registered with official bodies, this will be replaced by the correct name. In this way, the customer’s address will always contain the name of the correct and officially registered company.
  • Allow duplicate VAT: This option allows the registration of different customers with the same VAT numbers.
  • Skip API system fail: This option allows you to skip validation in case the VIES validation system fails during its verification process. It rarely happens that the VIES system fails, but it is a long shot.
  • Save in uppercase: The system saves the VAT number with its ISO code in uppercase when a customer registers a valid address.
    Blacklist: Set VAT numbers on a blacklist to disallow the registration of customers with some of these numbers. Write one by one separated by commas.

Order status management

The module creates two predefined order statuses during the installation phase:

  • VAT number pending validation
  • VAT number validated

In this section you can enable the option to display these order statuses when a customer places an order and one of these two situations occurs:

  • Order status when VAT is validated: When a customer has placed an order and their VAT number has been validated correctly. In this case, the configured order status will be displayed at the beginning and then the predefined order status for the payment method used by the customer will be displayed.
  • Order status when the API system fails: When a customer has placed an order and their VAT number could not be validated due to a failure in the API validation system and the «Skip validation» option is activated in the «Validation Settings» section. In this case, the customer will be able to continue with the purchase process and will be able to place the order, however, the status defined in this option will be displayed to notify the customer that their order is pending validation of their VAT number when service is restored. You must perform the validation of the VAT number manually from the «Customer VAT management» section and once the validation has been completed, manually change to another desired order status.

You can also use any available order status to use in either of these two situations by selecting it from the dropdown available in each option.

Customer group assignation by country

In this section you can select the allocation of customer groups by country. If you select the value «disabled» no group will be assigned to the client.

Customer group assignment is only made when a customer creates a new address or modifies a previously created address and that address contains a system-validated VAT number.

If a customer has several addresses from different countries with valid VAT numbers, the system will assign as many groups as you have configured by country in this section. As soon as the client eliminates the address and does not have more addresses related to the same country, the system will eliminate it from the group corresponding to that country.

  • Use as default group: If this option is activated, the group assigned to the client will be configured as the default group of this client. If the client adds more addresses with valid VAT numbers from different countries, the default group will change to the one corresponding to the last added address. The default group will be eliminated and changed to the next validated address whenever the client eliminates an address corresponding to a country and does not have more addresses from the same country.
  • Change default group with address: This option allows you to change the default customer group associated with the country in this section when the customer selects a default address or during checkout. If the selected address does not have a valid VAT number, the system will switch to the default customer group that has been configured in Prestashop customer settings (By default it is in «Customers» group). This option only changes the default client group, it does not assign or remove the client from the groups to which it is assigned.
  • Remove customer from another groups: When the customer creates a specific address with a valid VAT number, the system will assign them to the customer group selected by the country of their address and in turn will remove them from the rest of the groups where they were already assigned, including the Prestashop default group «customers». It will also assign the group to which it has been assigned as the default group.

Practical example for a better understanding of the options

Client: John DOE

Conditions

  • Group assignment by country Germany: Selected with the group «VAT Customers from Germany»
  • Group assignment by country Spain: Selected with the group «VAT Clients of Spain»
  • Other group options by country «Disabled»
  • Option «Use default group» activated.
  • Default customer group in Prestashop for customers: «Customers»
  • Option «Remove client from other groups» disabled.
  1. The customer registers for the first time in the store and automatically already belongs to the «Customers» group and this same group is selected by default.
  2. The customer creates a new address in Germany with a valid VAT number. The system adds you to the group «VAT Customers from Germany» and selects this group as the default group. Now the customer belongs to two groups: «Customers» and «German VAT Customers».
  3. The customer modifies this address and changes the country to Sweden. The system removes you from the «German VAT Customers» group and returns you to the default «Customers» group, which is the only group you are assigned to.
  4. The client adds two more addresses, one from Germany and later another from Spain. The system adds it to the group «VAT Customers from Germany» and «VAT Customers from Spain» and assigns it to the last created default group «VAT Customers from Spain».
  5. The client eliminates the address of Spain. The system removes it from the «VAT Clients of Spain» group and assigns it to the default group «VAT Clients of Germany» since it is the next address that has been created.
  6. The client decides to remove all addresses. The system removes it from all assigned groups and reassigns it to the default Prestashop group «Customers» so that it continues to belong to a default group.

In any case, and as long as the «Remove customer from other groups» option is not activated, the customer will always belong to the group that was previously assigned and is not one of those configured by country in the module. The system will never alter the assignment of that group. Only assign or deallocate the group selected by country in this module.

VAT exemption settings

  • VAT exemption: Activate this option if you want taxes to be eliminated both in the display of prices and in the calculation of prices in the shopping cart for customers with valid VAT numbers.
  • Address Type: Select the type of address for tax exemption. The options are «Delivery address», «Billing address», «Billing and delivery address». The client will be exempt from taxes if he has a valid VAT number in the type of address selected, or in one of them in the case of selecting both types of addresses.
  • Local country: This option allows you to assign the local country where prices and purchases are taxed regardless of whether you have a valid VAT number or not. The module does not validate the VAT number of customers with addresses corresponding to the local country.
  • Exemption of customer groups: This option allows you to eliminate taxes for certain groups of customers, even if the VAT number is valid or not. Leave it empty to disable it for all customer groups. If you check any groups, the tax exemption will only apply for customers in the selected groups. This option is completely independent of the [VAT exemption] option

Invoice settings

  • Invoice Legal Text: Insert content to be displayed within the legal text section of the invoice, only in invoices related to VAT exemption. The field is multi language.
  • Invoice note: Insert content to be displayed within the invoice note section, only in invoices related to VAT exemption. The field is multi-language and with a rich text editor.

VAT field settings

  • VAT field: You can make this field mandatory or optional.
  • Show only for company: If this option is activated, the VAT number field will remain hidden until the company name field is filled.
    If you have selected the field as mandatory and have the option to show only for the company activated, when the VAT number field is hidden, it will become optional.
  • Label: Insert the content that will be displayed as a label in the VAT number field. This field is multi language.
  • Legend: Insert the content that will be displayed as a description/legend in the VAT number field. This field is multi language.

These options may not work if you have installed a module that manages the checkout page. This happens with most checkout modules. However, there are some modules that can support some of these options (Example: SUNNYTOO.COM Easy Checkout Module or Presteamshop One Page Checkout Prestashop Module). If the module being used to manage the checkout page has options for the VAT number field, you must use the module’s own settings.

Brexit settings

  • Brexit: Activate the option to comply with the Brexit regulations to apply taxes at the point of sale on purchases equal to or less than £135 if the customer has a valid UK VAT number.
  • Business VAT Exemption: This option allows businesses with a valid VAT number to be tax exempt for purchases equal to or less than 135GBP.
  • Do not allow orders: This option enables the store to not allow orders whose amount in the shopping cart exceeds or is less than 135 pounds. Select «Disabled» to disable this option.

This system works in accordance with the Brexit regulations. If a customer wants to make a purchase with a United Kingdom address with a valid UK VAT number, if the total amount of the cart (Only your products) is equal to or less than 135 pounds, then the store will apply the taxes corresponding to that country. If the customer adds products to the cart and exceeds the threshold of 135 pounds, then the system will remove the taxes for that purchase.

The system calculates the total of the shopping cart in real time each time the customer updates their shopping cart.

For this option to work, the currency of the United Kingdom must be installed through the corresponding option in the BO, currency section (it does not have to be activated in the FO, it is enough to have the currency installed and the exchange rate updated) . To have the exchange rate updated, it can be done through the «Currency exchange rates» section on the module configuration page.

During the installation phase of the module, the country of United Kingdom will be configured so that it shows the provinces that will previously be inserted by the module. About 232 provinces will be inserted. In this way, the system ensures that when a customer buys from Northern Ireland, which belongs to the United Kingdom, the general system of intra-community purchases is applied, like the other countries of the European Union.

VOEC (Norway) settings – VAT On Ecommerce

  • VOEC: Activate the option to comply with the VOEC regulation regarding sales to consumers and companies residing in Norway.
  • VOEC mode: This option allows you to choose between two modes in order to facilitate VOEC operations with consumers residing in Norway.
    Standard mode: Prohibits processing the order if a customer has added to the cart products that fall within the VOEC regulation (amount less than NOK 5,000) together with products that are outside the VOEC regulation. Notify the customer that they have to remove the products that are out of regulation from the cart.
    Bunding mode: Allows a customer to complete the purchase process even if they have a combination of products that fall within the VOEC regulation and others that are outside the regulation in their shopping cart.

This system works according to the VOEC regulations. Sales to consumers residing in Norway will be charged Norwegian VAT at the point of sale, as long as they purchase products that are regulated by the VOEC (Amounts less than NOK 5,000). In the case of companies, if they provide a valid Norwegian number, they will be able to make purchases exempt from VAT.

The system calculates the total of the shopping cart in real time each time the customer updates their shopping cart.

For this option to work, the Norwegian currency must be installed through the corresponding option in the BO, currency section (It does not have to be activated in the FO, it is enough to have the currency installed and the exchange rate updated). To have the exchange rate updated, it can be done through the «Currency exchange rates» section on the module configuration page.

During the installation phase of the module, the country of Norway will be configured to show the provinces that will previously be inserted by the module. In this way, the management of orders to Norway is facilitated.[/bs_notification]

 

Email settings

  • Email addresses: You can configure different email addresses to receive email from this module. You must enter a correct email address. For multiple addresses, you must write them separated with commas (Ex: test@test.com, test2@test2.com…).
  • API Alert Email: Send an alert email to the selected addresses, when the customer registers and the VAT validation by the API system fails.
  • Send email to client when API fails: Sends an email to the customer when they have placed an order with an address created where VAT number validation failed due to API system failure and the «Skip validation» option from the «Validation Settings» section has been activated.
  • Customer validation email: Send email informing when the customer registers and the VAT validation by VIES or GOV.UK is successful.

CRON settings

In this section you will be able to configure the options to execute with the CRON task of analyzing addresses and validating customer VAT numbers. This CRON task is exactly the same as the module scanning system in the «VAT Management» section. It is very useful to schedule the scanning system automatically and on a daily basis to ensure that customers continue to keep their VAT number valid at all times. In this way, the merchant ensures that the tax-exempt purchase operations are carried out in accordance with the regulations on intra-community operations with a valid VAT number.

This scheduled task will update the list of customers in the «VAT Management» section.

In the event that the CRON task process is interrupted for some reason or you want to start the scan from the last address previously scanned, activate the option «Continue the scanning process from the last scanned address». The system will start the scan from the last scanned address and not from the beginning.

It is recommended that you run this scheduled task at night or during off-peak hours in the store to avoid performance issues.

 

Currency exchange rates

In this section you can view the exchange rates of all the currencies available in the store and you can update their exchange rates. In addition, the system has a URL to configure the scheduled updating of currency exchange rates automatically through a CRON task. This option is very useful if you are using the Brexit and VOEC option.

Module Sections in Admin Panel

Admin Menu Tabs

When the module is installed, the system incorporates some sections in the administration panel:

  • Customer VAT Number Management: This section shows a list of customers who have registered an address and the result of the validation of the VAT and company fields. Actions can also be carried out such as scanning addresses, sending emails to clients to notify them that they have an invalid VAT number, checking the VAT number in official systems, etc…
  • Customer VAT Exemption Management: This section shows a list with the customers who have been granted manually, who are exempt or forced to paying taxes in the store.
  • Order management: This section shows a list of the orders made by customers who have a valid VAT number and are exempt from taxes or belong to Brexit.
  • VAT Check Tool: With this tool you can make queries about the VAT numbers to the platforms of official bodies and thus check if the VAT number is valid and know details such as the name of the company registered with that number.

Customer VAT Management

Customer VAT Management

In this section, on the one hand we have the list of all the clients that have addresses that meet the criteria (countries for validation) to be able to have a VAT number, and on the other hand we have a scanning system that checks all the addresses stored in the database and that meet the same criteria and performs the validation of their VAT numbers.

Every time a customer registers an address that meets the VAT number validation requirements, it will be automatically inserted in this list with all the details indicated in the table.

A statistics table is also available so you can view the total numbers of addresses that have a valid or invalid VAT number and company field.

If the module has been installed for the first time, a first scan must be carried out to validate the VAT numbers and insert them in the list.

Customer VAT list

In this list we can obtain, among others, the following useful information for the client:

  • Customer Name: This is the name of the customer registered on the address form
  • Address: The customer’s address.
  • Country: The client’s country of registration.
  • Default Client Group: The default group that the registered client belongs to.
  • VAT number: The registered VAT number.
  • VAT Validation: Shows an icon to know if the VAT number is valid or not after the validation process.
    Invalid VAT
    Valid VAT
    VAT number validated by default of the validation system by failure of the API system when option «Skip validation» is enabled.

  • Company validation: Shows an icon to know if the company name has been validated or not (If the Company Validation option is activated).
    Invalid company name.
    Valid company name.
    Company name not verified by the system (When the Company Validation option is disabled).
  • System check: Shows an icon to find out if the API validation process has worked correctly or there has been a connection problem with the system to validate the VAT numbers and company name.
    Failed API validation process
    Successful API validation process
  • Status: Information on the status of the VAT number and the validation process.
  • Registration Date: The date and time when the customer has registered that address.

Status

Once the validation of the VAT number is carried out, it can present different statuses:

  • Invalid VAT number format: This means that an attempt was made to validate the VAT number format but it did not pass the format validation process.
  • Format validation: The VAT number has successfully passed the format validation process.
  • VAT number empty but VAT number field is optional: It has been detected that a VAT number has not been registered and the field is empty, but depending on the configured option, the field is optional. Therefore it passes the validation successfully.
  • VAT number empty: The VAT number field is empty and the field is required. In this case the validation does not pass.
  • [HTTP code (response code number) | VIES/GOV.UK 1-Way Validation]: Displayed when the chosen validation system type is VIES/GOV.UK 1-Way. The response code is displayed and ends with a message with the validation result.
  • [HTTP code (response code number) | VIES/GOV.UK 2-Way Validation]: Displayed when the chosen validation system type is VIES/GOV.UK 2-Way. The response code is displayed and ends with a message with the validation result.
  • The VAT number is duplicated: Indicates that there are other different customer addresses that have the same VAT number. In this case, it is validated even if it is a duplicate as long as the option to allow duplicate VAT numbers has been selected.
  • The company name (company name) does not match the company name registered with VIES/GOV.UK: The company name has not been validated because it does not match the name registered with official bodies.
  • The company name field is empty: The company name field is empty and must be indicated, for that reason it could not be validated correctly.
Options within the listing

A series of actions can be performed on each record in the list, just as they can be performed in bulk when more than one record is selected.

  • View customer details: Redirects to the customer details page.
  • Check VAT on VIES/GOV.UK: Checks the VAT number through the official APIs and displays the result in a popup window.

  • Validate VAT manually: Validates the VAT number manually indicating that it is valid without going through any validation process.
  • Validate VAT with VIES/GOV.UK: Performs the validation process using the VIES or GOV.UK system (in the case of a UK VAT number) and displays the result, validating the number or not.
  • Send VAT validation request email: Sends an email to the customer requesting that they have to insert a valid VAT number in the address so that it can be validated.
  • Remove VAT: Empty the VAT number field for that address.
  • Delete: Delete the record from the database.
Edit and validate the VAT number

You can manually modify the VAT number registered by the customer and validate it or not manually. The validation process is manual, that means that it will be valid or not depending on the option you choose without going through any format validation process or VIES.

If the VAT number is changed, it will also be changed from the customer address section.

If the address is removed from the addresses section, the record for that address will also be removed from this listing.

Customer VAT Scanner

The function of the scanner is to carry out a complete scan of the database and extract the addresses of the clients that meet the conditions so that they can or must have the VAT number registered. It depends on the addresses that correspond to the countries chosen for validation.

During the scanning process, the validation process of the VAT number inserted in the address is also executed using the method selected in the validation options from the module configuration page.

Scan options

Before performing the scan, a pop-up window will open to choose the options that you want the system to perform during the scan:

  • Continue the scanning process from the last scanned address: The system will start the scanning process from the last address scanned. This is very useful if for whatever reason the previous scan ended before the process was complete or if you want the system to start the scan from the last address instead of the beginning.
  • Skip API system fail: If you enable this option, the system will skip address VAT number validation when the VIES/GOV.UK API system fails and will give the VAT number as valid. In the system status, the icon will appear in red warning that there was an error during validation.
  • Delete address with empty VAT number field: Delete the customer address that has an empty VAT number field.
  • Delete address with invalid VAT number field: Delete the customer address that contains an invalid VAT number.
  • Delete address with duplicate VAT number: Delete the customer’s address that has a duplicate VAT number. The system is smart and only removes addresses that have entered a duplicate VAT number, keeping the addresses of the first original customer who first registered with that VAT number.
  • Send email to customers with invalid VAT number field: Send an email to customers who have the invalid VAT number on file.
  • Send email to customers with empty VAT number field: Send an email to customers who have the VAT number field empty.
  • Email customers with duplicate VAT number field: Send an email to customers who have a duplicate VAT number on file with another customer.
  • Assign customers with a valid VAT number to the group selected by country: Assigns the groups selected by countries in the «Customer group assignment» section to customers who have a valid VAT number.
  • Complete the company field in the address with the company registered in VIES or GOV.UK when the VAT number is valid: During the scanning process, the system will insert in the customer’s address, the company name that is registered in official bodies according to the indicated VAT number.
  • Email customers with an invalid business name on file: Send an email to the customer with the invalid business name. This option will be available if the «Company validation» option is activated in the module configuration.
  • Send email to customers with empty company name: Send an email to the customer who has the empty company name field. This option will be available if the «Company validation» option is activated in the module configuration.

Once the scanner is done, a button will appear to refresh the page and thus be able to view the updated list. You can also download the excel list of customers with VAT numbers from a button.

Depending on the number of addresses stored in the database, the scanning process could take a few minutes, even hours. We recommend that in the case of having thousands of addresses stored in the database, the scanning process is run at appropriate times with little activity in the store, to avoid performance problems.

Customer Exemption Management

Customer VAT Exemption Management

In this section, customers can be added manually to whom the tax exemption or imposition will be applied, regardless of whether they have an address with a valid VAT number or not, or meet the other conditions. This configuration will always have priority over the rest.

If a customer is added to this list and qualified for tax exemption. Both the prices of the products and the orders can be made exempt from taxes in any condition, and vice versa in the event that you are forced to pay taxes.

This is very useful when you want to apply the exemption or the obligation to pay taxes to certain customers who meet a series of conditions or by express wish.

If you have activated the multi-store mode, you can add the exemption or obligation to pay taxes to the customer in certain stores or for all stores.

When the client is included in this list, one of the conditions chosen for the payment of taxes will be applied to him in a forced manner and as a priority with respect to the other options configured in the module. If you want the customer to apply the tax exemption according to the general conditions of the module (Have a valid VAT number and belong to the European Union) you must remove it from this list

Orders Management

Orders Management

In this section they manage all the orders that are made in the store and are subject to tax exemption for having a valid VAT number or are subject to the Brexit or VOEC regulation.

It contains a section for statistics, showing the total amounts of sales, both those that are exempt from taxes and those subject to Brexit regulations, with the function of filtering by dates.

The options available to view each record are:

  • View order: View the order data.
  • Download invoice: You can download the invoice related to this order as long as it has been generated.
Export of invoices

In this section you can massively download the invoices generated by these orders by date ranges. The date range refers to the creation of the order and not to the invoice generation date, which may be different from the previous one.

Orders importation

In this section you can import the orders that were already saved in the database and have not yet been recorded in this list.

The import process could take a few minutes if there are thousands of saved orders in the database. It is recommended to increase the max_execution_time server value to 300 or higher

VAT check Tool

VAT Check Tool

In this section, a VAT number verification tool is made available that connects directly to the official VAT verification system APIs of VIES, GOV.UK (for UK VAT numbers) or Norway.

You must insert the VAT number to check and press the «Check VAT» button. The module system will connect via API with the official systems to check if the inserted VAT number is valid or not and will display a table with the data related to this number (country, registered company name, company address, ISO code of the country, validity of the VAT number and date and time of the request)

There is no limitation on the number of queries to be made and the service is completely free.

Tax exemption system

Once the customer registers an address with a valid VAT number, the system will remove the tax calculation and display the prices without tax in the following places:

  • Product sheet
  • Product List
  • Shopping cart
  • Cart Summary
  • Order details in the Back-office
  • Purchase invoices
  • Corrective invoices

The tax exemption system works even with discounts, cart rules, specific prices, etc…

It may be the case that the customer has two addresses, one with a valid VAT number and one without a VAT number. As long as the customer chooses or has predefined the address with the valid VAT number as the shipping address, the system will eliminate the taxes in the prices.

If a customer already had a registered address with an invalid VAT number before installing the module. The system will notify you in a warning message when you go to make the purchase or when you access the section of your addresses.

Invoices and order slip invoices

Invoices issued on orders with registered customers with a valid VAT number will be exempt from taxes and are not shown in the invoice breakdown.

Tax exempt invoice

 

Brexit regulated invoice

In the invoices where the client is exempt from taxes, several details are shown:

  • «Exempt from VAT in accordance with section 259B of the general tax code.»
  • Custom note configured in the «Invoice Settings» section in the module options.
  • Custom legal text configured in the «Invoice Settings» section in the module options.
  • In the case of invoices regulated by Brexit, if they are exempt from taxes, the message «Reverse charge: client to account for VAT to HMRC» will be displayed to comply with the regulations that regulate operations within the framework of Brexit.
  • In the case of the VOEC regulation, it will be shown on the invoice that the tax collection is regulated by the VOEC, for consumers residing in Norway.
